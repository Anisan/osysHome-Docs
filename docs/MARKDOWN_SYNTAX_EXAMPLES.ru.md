## Текстовое форматирование

- **Жирный текст** — `**жирный**` или `__жирный__`
- *Курсив* — `*курсив*` или `_курсив_`
- ***Жирный курсив*** — `***текст***`
- ~~Зачёркнутый~~ — `~~зачёркнутый~~`
- `Инлайн-код` — обратные кавычки
- Нижний индекс — `H<sub>2</sub>O` → H<sub>2</sub>O
- Верхний индекс — `x<sup>2</sup>` → x<sup>2</sup>
- Подчёркивание — `<ins>вставленный текст</ins>` → <ins>вставленный текст</ins>

## Переносы строк

В `.md` файлах перенос создаётся одним из способов:

- Два пробела в конце строки  
  следующая строка

- Обратный слэш в конце строки\
  следующая строка

- Тег `<br/>` в конце строки<br/>
  следующая строка

Пустая строка между абзацами создаёт новый параграф.

## Mermaid-диаграммы

### Блок-схема (Flowchart)

```mermaid
flowchart LR
    A[Старт] --> B{Условие?}
    B -->|Да| C[Действие 1]
    B -->|Нет| D[Действие 2]
    C --> E[Конец]
    D --> E
```

### Диаграмма последовательности

```mermaid
sequenceDiagram
    participant U as Пользователь
    participant D as Docs
    participant S as Поиск
    U->>D: Открыть документ
    D->>D: Загрузить MD
    D->>S: Индексировать
    S-->>U: Показать контент
```

### Диаграмма классов

```mermaid
classDiagram
    class BasePlugin {
        +title
        +description
        +admin()
        +initialization()
    }
    class Docs {
        +_docs_index
        +_build_docs_index()
        +_render_markdown_doc()
    }
    BasePlugin <|-- Docs
```

### Диаграмма состояний

```mermaid
stateDiagram-v2
    [*] --> Idle
    Idle --> Scanning: refresh
    Scanning --> Indexing: scan done
    Indexing --> Ready: build done
    Ready --> Idle: clear cache
```

### Ганта

```mermaid
gantt
    title План разработки
    dateFormat  YYYY-MM-DD
    section Фаза 1
    Анализ требований    :a1, 2024-01-01, 7d
    Проектирование       :a2, after a1, 5d
    section Фаза 2
    Реализация           :b1, after a2, 14d
    Тестирование         :b2, after b1, 7d
```

### Entity Relationship

```mermaid
erDiagram
    DOC ||--o{ SECTION : contains
    DOC {
        string id
        string title
        string path
    }
    SECTION {
        string id
        string heading
    }
```

## Сноски

Синтаксис сносок[^1] позволяет добавлять примечания внизу документа.

Сноска может быть многострочной[^2].

[^1]: Текст сноски.
[^2]: Многострочная сноска.  
Добавьте два пробела в конце строки для переноса.

## Модели цветов

В инлайн-коде можно указать цвет в форматах HEX, RGB, HSL — некоторые рендереры показывают визуализацию цвета:

- `#DA690A` — HEX
- `rgb(9, 105, 218)` — RGB
- `hsl(212, 92%, 45%)` — HSL

## Экранирование

Обратный слэш `\` перед спецсимволом отключает форматирование:

Специальные символы: \* \_ \` \[ \] \( \) \# \. \!

Пример: переименуем \*старый-проект\* в \*новый-проект\*.
